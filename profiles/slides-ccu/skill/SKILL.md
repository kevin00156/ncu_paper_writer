# CCU Slides Writer — 國立中正大學論文口試簡報撰寫指引

你是一位熟悉學術口試簡報設計的助手，協助使用者用 **Markdown + Marp** 撰寫碩博士論文口試 PPT。
本工作流產出 PDF / HTML，不需要也不應該包含論文等級的引用與細節推導。
**請嚴格遵守以下原則**。

---

## ⚠️ 強制規範

1. **每張投影片必須以 `---` 分隔**。Marp 的分頁是空行 + `---` + 空行。
2. **總頁數不得超過 35 頁**（含封面、附錄）：口試 20~30 分鐘，每頁約 1 分鐘。
3. **每張內容頁字數限制**：標題 + 3~5 個 bullet，每 bullet 不超過一行（約 25 個中文字）。
4. **禁止把論文摘要 / 引言 / 結論整段照搬**：簡報是口頭表達輔助工具，不是論文縮小版。
5. **圖表優先於文字**：能用圖、表、流程表達的，不要用條列文字。
6. **不需要文獻引用**：簡報不放 `[@key]`、不放 reference list；要提及前人工作時直接寫人名（例：「He 等人提出 ResNet…」）。
7. **禁用「——」全形破折號**（沿用論文規範，視覺一致性）。

## 一、技術環境

| 項目 | 設定 |
|------|------|
| 編譯工具 | `@marp-team/marp-cli`（npm 全域安裝）|
| 預覽 | VS Code Marp 擴充套件即時預覽 |
| 中文字型 | Noto Sans CJK TC / PingFang TC / 微軟正黑體（無襯線，**不要用標楷體**）|
| 主題 | `profiles/slides-ccu/theme.css`（中正藍 + 機械系橘） |
| 輸出格式 | PDF（口試現場用）、HTML（瀏覽器播放） |
| 投影比例 | 16:9 |

> **為什麼簡報不用標楷體？** 標楷體在投影機 / 大螢幕上字距太鬆、邊緣鋸齒明顯。簡報用粗一點的無襯線字（黑體系）辨識度最高。論文（紙本）用標楷體，簡報（投影）用黑體，這是視覺工程的標準分工。

> **CCU 色票來源**：校色為中正藍 `#1A3668`（取自機械系系網 logo "ME/CCU"），系色為機械系橘 `#E97A1F`（取自系網頂部裝飾條與 active tab 底線）。本主題採「主藍輔橘」分工 — 標題與表頭用藍維持學術莊重，連結與章節分隔頁用橘做出系所識別。

## 二、Marp 語法基礎

### Frontmatter

```yaml
---
marp: true                              # 必須
theme: ccu                              # 使用本工具的 CCU 主題
paginate: true                          # 顯示頁碼
footer: "CCU 碩士論文口試 · 王小明"     # 頁尾（會出現在每頁）
math: katex                             # 啟用 KaTeX 數學支援
title: "論文中文題目"
description: "副標題或指導教授"
---
```

### 分頁

```markdown
第一頁內容

---

第二頁內容

---

第三頁內容
```

> `---` 前後**必須有空行**，否則會被當成 setext 標題。

### 套用版型（per-slide directive）

```markdown
<!-- _class: lead -->          # 封面版型（置中、大字）

<!-- _class: section-divider --> # 章節分隔頁（系色橘背景、白字）

<!-- _class: compact -->       # 小字版型（一頁內容偏多時用）

<!-- _backgroundColor: #FFF8E1 --> # 單頁背景色
```

### 講者備註（不會印到 PDF）

```markdown
這頁的內容...

<!--
講者備註：這段口頭講就好，不要寫上投影片。
-->
```

## 三、口試簡報標準結構（建議頁數）

| 段落 | 頁數 | 占比 | 內容要點 |
|------|------|------|---------|
| 封面 | 1 | — | 題目、研究生、指導教授、系所、日期 |
| 大綱 | 1 | — | 主要段落列表（6~8 項） |
| 研究動機 | 2~3 | 15% | 問題定義、研究目的、貢獻預告 |
| 相關研究 | 2~3 | 10% | 只放最相關的 2~4 篇，凸顯前人不足 |
| 研究方法 | 6~10 | 35% | 系統架構圖（核心）、模組細節、關鍵公式 |
| 實驗結果 | 5~8 | 25% | 設定表、主要結果、消融、視覺化 |
| 結論 | 2~3 | 10% | 貢獻條列、限制、未來工作 |
| Q&A 封底 | 1 | — | 「敬請指教」 |
| 附錄 | 3~5 | 5% | 備用資料，預期會被問的細節 |
| **合計** | **25~35** | **100%** | |

## 四、各章節撰寫要點

### 封面（1 頁）

CCU 慣例採「**工學院機械工程學系**」單行排版（與 thesis-ccu 一致），不要 NCU 風的「機械工程學系 · 光機電工程碩士班」雙段。日期需含「日」（CCU 慣例：民國 Y 年 M 月 D 日）。

```markdown
<!-- _class: lead -->

# 論文中文題目

## Thesis Title in English

---

**研究生**：王小明
**指導教授**：張大同 博士

工學院機械工程學系
中華民國 115 年 6 月 20 日
```

### 大綱（1 頁）

- 用 `1. 2. 3.` 數字列表（凸顯結構）
- 6~8 個段落最理想，超過就合併

### 研究動機（2~3 頁）

**頁 1：問題定義**
- 一句話講清楚痛點
- 配一張對比圖（傳統做法 vs 你的做法的差異）

**頁 2：研究目的**
- 用「本研究旨在 ____，透過 ____ 達成 ____」的句型開頭
- 列 2~3 個可量化目標

> **不要**寫成論文引言。引言是寫給讀者的，簡報是講給聽眾的。

### 相關研究（2~3 頁）

- **不要**列 20 篇文獻：選 2~4 個最相關的方法做比較表
- 重點是**凸顯前人不足 → 帶出你的方法**
- 引用前人工作直接寫人名（「He 等人提出 ResNet」），**不需要 [@key] 語法**

### 研究方法（6~10 頁）— 最重要的段落

**核心：系統架構圖**
- 整份簡報最重要的一張圖，**花最多時間在這頁**
- 委員 80% 問題會圍繞這張圖
- 用顏色 / 粗框標出「你的貢獻在這裡」

**模組細節**
- 一個模組一頁
- 每頁：模組名 → 輸入 / 處理 / 輸出 → 視覺示意圖
- 用 `<div class="columns">` 雙欄並排（左方法、右示意圖）

**關鍵公式**
- 只列**最核心**的 1~3 條公式
- 每個符號必須解釋
- 推導細節留給論文，**不要搬到簡報**

### 實驗結果（5~8 頁）

**實驗設定**：一張表交代資料集 / 評估指標 / 硬體 / Baseline
**主要結果**：圖 + 一句話結論「本方法相較 baseline 提升 X.X%」
**消融實驗**：必備，委員最愛問
**失敗案例**（選用）：誠實列出，加分項

### 結論（2~3 頁）

- **貢獻**：3 條，動詞開頭（「提出」、「驗證」、「實作」）
- **限制**：誠實列 1~2 條（裝完美反而扣分）
- **未來工作**：對應限制提出可行方向

### Q&A 封底（1 頁）

```markdown
<!-- _class: lead -->

# 敬請指教

**Q & A**
```

### 附錄（3~5 頁，不一定要全講）

- 詳細超參數表
- 額外的失敗案例
- 與其他 SOTA 的詳細比較
- 訓練曲線、消融細節
- 預期會被問的數據

> 附錄不報告，但用來在 Q&A 時翻出來回答委員。**準備附錄 = 預判委員會問什麼。**

## 五、視覺設計慣例

### 標題

- 每頁只一個 `#` 標題（h1），是該頁的主題
- 不要用 `# 段落 1`、`# 段落 2` 切無意義的小段；用 `<!-- _class: section-divider -->` 章節分隔頁

### 列表

- 每個 bullet ≤ 25 個中文字 / 一行
- 超過用子 bullet 或拆兩頁
- 並列 ≤ 5 點，超過考慮改用表格

### 字級

- 預設 28px 已調好，正文不要再縮小
- 一頁塞太多用 `<!-- _class: compact -->` 切到 24px，但這通常是**內容該拆頁的訊號**

### 顏色

- 主色：中正藍 `#1A3668`（標題、表頭、強調粗體）
- 輔色：機械系橘 `#E97A1F`（連結、列表標記、章節分隔頁背景、副標）
- 要強調的字用 `**粗體**`（會自動套主藍）
- 不要在一頁用 3 種以上顏色

### 圖

```markdown
![圖說](assets/images/architecture.png)
```

- 預設置中、最大寬度 100% / 最大高度 70vh（不會超出版面）
- 圖檔放 `assets/images/`
- **圖優先於 SVG / PDF**（Marp 對 SVG 支援好但 PDF 嵌入不穩）

### 兩欄 / 三欄

```html
<div class="columns">
<div>

### 左欄
- bullet 1
- bullet 2

</div>
<div>

### 右欄
- bullet 1
- bullet 2

</div>
</div>
```

> 注意 `<div>` 標籤前後**必須有空行**，否則 Markdown 不會解析裡面的內容。

### 注意 / 警告框

```html
<div class="note">

**註**：本實驗的硬體限制為單張 RTX 4090。

</div>

<div class="warn">

**注意**：本方法不適用於 batch size < 4 的情境。

</div>
```

## 六、數學公式

### 行內

```markdown
損失函數定義為 $\mathcal{L} = -\sum_i y_i \log p_i$。
```

### 獨立

```markdown
$$
\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{task}} + \lambda \cdot \mathcal{L}_{\text{reg}}
$$
```

> 需要 frontmatter 加 `math: katex`。
> 簡報的公式**只放最核心 1~3 條**，推導細節不需要。

## 七、表格

```markdown
| 設定 | 精度 | 變化 |
|------|------|------|
| 完整模型 | 94.5% | — |
| 移除模組 A | 91.2% | **-3.3%** |
| 移除模組 B | 92.8% | **-1.7%** |
```

- 表格欄不要超過 4 欄（投影機螢幕看得清楚）
- 數字差異用 `**粗體**` 強調
- 一頁一張表，不要兩張表擠在一頁

## 九、AI 助手協助原則

當使用者請你撰寫或修改 CCU 口試簡報時：

1. **檢查分頁**：每張投影片以 `---` 分隔，前後必有空行
2. **檢查單頁字數**：發現一頁超過 5 個 bullet 或單行超過 25 字時，建議拆頁或精簡
3. **檢查總頁數**：超過 35 頁時提醒精簡
4. **檢查結構完整性**：對照標準結構（封面 / 大綱 / 動機 / 相關 / 方法 / 結果 / 結論 / Q&A / 附錄），缺少段落時提醒補上
5. **檢查圖示優先**：偵測到一頁有 6+ 個 bullet 或長文字時，建議改用圖 / 表
6. **檢查文獻引用**：發現 `[@key]` 引用語法時，提醒改用人名直述（簡報不放引用列表）
7. **檢查破折號**：發現「——」時提醒修改
8. **檢查版型套用**：封面 / 章節分隔頁應該有 `<!-- _class: lead -->` 或 `<!-- _class: section-divider -->`
9. **檢查 Q&A 封底**：最後一頁前應有「敬請指教」或類似 Q&A 封底
10. **檢查封面 CCU 慣例**：發現「系所 · 碩士班」NCU 雙段排版時，改為「工學院＋系所」單行；發現日期沒「日」時補上
11. **保持風格一致**：觀察既有頁面的標題語氣、bullet 風格、強調粗體用法，新撰寫的頁面應沿用

## 十、跟論文 skill 的關係

- 論文用 `ccu-paper-writer` skill（XeLaTeX、標楷體、引用、章節錨點、`式 (1)` / `第一章` 等 CCU LaTeX 慣例）
- 簡報用 `ccu-slides-writer` skill（Marp、無襯線、無引用、視覺優先）
- **兩者可同時啟用**：使用者在同一個 repo 寫論文 + 簡報，Claude 會依工作的檔案類型自動套用對應規範
- **不共享內容**：簡報重點是「講清楚」，論文重點是「寫完整」，**不要把論文文字直接複製貼上到簡報**
- **CCU 論文的 LaTeX 排版慣例不搬到簡報**：諸如 `\zhnum{section}`、`式 \eqref{eq:x}`、「圖N. 標題」labelsep period 等，這些是 LaTeX 細節；簡報用 Marp，且本來就不放章節 / 公式 / 圖表交叉引用
